## 배운 내용

### 스칼라곱
- 4 * v1 : v1의 방향은 같고 크기만 4배
- 각 원소에 4씩 곱해짐

### 원소별 연산 (element-wise)
- v1 + v2 : 같은 자리끼리 더해짐 (성분별 연산)
- v3 = 4 * v1 - 2 * v2 : 스칼라곱 + 원소별 뺄셈이 한 줄에

### 리스트 vs np.array
- 리스트 + : 이어붙이기 (나열) → [2,5,4,7] + [4,1,0,2] = [2,5,4,7,4,1,0,2]
- np.array + : 벡터합 (성분별 덧셈)
- 그래서 벡터 연산엔 np.array를 씀

### transpose
- .T : 행 <-> 열 뒤집기 (행벡터를 세로 열벡터로)
- 대괄호 2개 [[...]] = 2차원이어야 transpose가 먹힘
- shape 변화: (1, 4) → (4, 1)

## Code Challenge - 스칼라·벡터 곱 시각화

### 핵심
- plt.plot(x들, y들) : 점 단위가 아니라 x좌표 모음 / y좌표 모음으로 받음
  - plt.plot([0, v[0]], [0, v[1]]) -> (0,0)에서 (v[0],v[1])까지 선 = 벡터
- np.random.randn() : 정규분포에서 랜덤 스칼라 하나 (대체로 -2~2, 부호 섞임)
- for 루프로 랜덤 벡터 10개를 겹쳐 그림

### 관찰
- sv = s * v (스칼라곱)이라 모든 빨간 벡터는 원본과 같은 직선 위
- s 부호에 따라 앞(양수)/뒤(음수)로만 갈릴 뿐, 직선(축)은 절대 안 벗어남
- 결론: 스칼라곱은 벡터의 직선을 안 바꾸고, 그 위에서 길이·방향(앞뒤)만 바꾼다

### 꾸미기
- plt.axis([-6,6,-6,6]) : 축 고정 (자동이면 매번 바뀌어 비교 어려움)
- plt.legend() : label 붙은 선의 이름표를 박스로 표시
